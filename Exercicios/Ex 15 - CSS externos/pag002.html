CSS inline

Oque é o CSS inline ?

O CSS inline é uma variação de desenvovimento como todas a outras, seu diferêncial é a forma como os comandos são inseridos, juntos as tags de HTML (semântica), com isso misturando semâmtica e estilo.

Esta não é uma forma muito comum de se desenvolver um site e nem muito util na verdade, por possuir mais contras que prós e muitas limitações, além de que polui e muito o código, oque não é nada bom.

Outro grande problema no inline é a falta de versatilidade, caso fosse colorir uma linha você teria que fazer a mesma coisa manualmente em todas as outras. Caso fosse ser necessário uma alteração muito grande no site você teria que fazer a alteração manualmente em todas as linhas.

Não vamos utilizar o modelo inline por tanto não espera grandes registros.

CSS local

Aqui veremos o estilo CSS local, que seria um método melhor que o anterior, neste veremos uma forma de desenvolver o estilo de um site de forma separada, de maneira que, deixará o documento mais organizado e o código também. Para fazer vamos abrir a tag "style" logo abaixo de title, em seguida utilizar metodo de seletor, então você irá abrir o body entre chaves e toda alteração será feita dentro dessas chaves. A grande vantagem é que por modelo de seleção todas as alterações são feitas de forma ampla, ou seja, se eu abrir o p {} todas as alterações de cor e forma serão feitas em todos os textos do documento de uma vez só, isso facilita o trabalha, deixa menos poluição e facilita a manunteção e alteração do código.

E as desvantagens ?

O estilo CSS local possui algumas desvantagens simples mas que podem ser bem incomodas, são elas:

  1. Ocupar muito espaço em relação ao conteúdo da página.
  2. Não abranger todas as páginas. (copiar o mesmo código sempre que for por em outra página.)
  3. Uma alteração severa no desenvolvimento de designe de uma site com documentos extenços, isso vai ser cruel.

Por fim concluímos que, o estilo local não é indicado para arquivos muitos extensos com 10,20,30 páginas, na verdade, qualquer documento com mais de umas 2 ou 3 páginas já vai se tornar chato. Mas esses pontos citados não tornam o método local ineficiente ou até mesmo absoleto, ele é sim um bom método a ser usado, para documentos menores por exemplo ele pode ser bem útil.
